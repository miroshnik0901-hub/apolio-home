# Как бот получает доступ к Google Sheets

## Аутентификация: сервисный аккаунт

Бот аутентифицируется через **сервисный аккаунт Google** — технический аккаунт без пароля, который работает через JSON-ключ. Ключ хранится в переменной окружения `GOOGLE_SERVICE_ACCOUNT` как base64-строка. Railway передаёт её боту при запуске.

Email сервисного аккаунта: `apolio-home-bot@apolio-home.iam.gserviceaccount.com`

При инициализации `sheets.py`:
1. Декодирует base64 → получает JSON
2. Через библиотеку `gspread` создаёт авторизованный клиент
3. Этот клиент — единственная точка входа для всех операций с таблицами

## Права доступа к таблицам

Каждая таблица должна быть **расшарена на email сервисного аккаунта** с правами редактора. Это делается один раз при создании бюджетного файла пользователя. Таблицы открываются по **Sheet ID** — строке из 44 символов из URL (`docs.google.com/spreadsheets/d/<ID>/`).

## Архитектура: SheetsClient

Класс `SheetsClient` в `sheets.py` оборачивает gspread-клиент и добавляет **кэш-слой**:

- **TTL-кэш** для транзакций — чтобы не читать 300 строк при каждом сообщении пользователя
- **Статический кэш** для курсов валют и конфигов — они меняются редко

Каждый бюджетный файл инкапсулирован во внутреннем классе `BudgetSheetClient`, который открывается по Sheet ID через env-переменные.

## Два режима чтения

### FORMATTED (`get_all_values()`)
Возвращает всё в виде строк:
- Дата `2026-04-08` → строка `"2026-04-08"`
- Булевое значение `True` → строка `"TRUE"`

Удобно для фильтрации по тексту и отображения пользователю.

### UNFORMATTED (`value_render_option='UNFORMATTED_VALUE'`)
Возвращает сырые типы Python:
- Дата → числовой серийник (например `46023`)
- Булевое значение → `True` / `False`

Нужен когда важны точные числовые значения — например, для формул в Google Sheets.

> ⚠️ Именно разница между этими режимами стала корневой причиной бага T-251: Summary-лист показывал нули, потому что формулы ожидали строку `"2026-01"`, а в ячейках хранился серийник `46023`. Фикс: заменить `LEFT(date, 7)` на `TEXT(date, "yyyy-mm")` в формулах.

## OAuth: создание новых таблиц

Для **создания новых файлов** (онбординг пользователя) используется отдельный OAuth-flow через переменные:
- `GOOGLE_OAUTH_CLIENT_ID`
- `GOOGLE_OAUTH_CLIENT_SECRET`
- `GOOGLE_OAUTH_REFRESH_TOKEN`

Сервисный аккаунт не может создавать файлы от имени пользователя — OAuth может. После создания файл автоматически расшаривается на сервисный аккаунт, и дальше работает стандартный путь через gspread.

## Схема потока данных

```
Telegram message
      ↓
   bot.py
      ↓
 SheetsClient (sheets.py)
      ↓
  TTL Cache ──── hit? → return cached
      ↓ miss
  gspread client
      ↓
  Google Sheets API (REST)
      ↓
  Google Sheets file
```
