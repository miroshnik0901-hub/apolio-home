# Menu & Dashboard Restructure — Design Doc
**T-112 + T-113 | 2026-04-09**

---

## Принцип: Бот = семейный бюджетный ассистент

Главное: добавить расход → посмотреть сколько осталось → посмотреть куда ушло.
Всё остальное — вторично. Меню должно отражать частоту использования, а не полноту функций.

---

## MENU v2

### Reply Keyboard (всегда видна, 3 кнопки)

```
[ 💰 Бюджет ]  [ ➕ Добавить ]  [ ☰ Меню ]
```

Без изменений — работает хорошо. "☰ Меню" открывает inline menu (fix T-108 уже сделан).

### Inline Menu — новая структура

**Уровень 0 (корень):** Плоский список, максимум 6 кнопок. Без подменю на первом экране.

```
[ 📋 Отчёт · этот месяц ]   [ ◀ Прошлый месяц ]
[ 📝 Последние записи   ]   [ 🔍 Найти...      ]
[ ⚖️ Баланс (кто кому)  ]   [ ⚙️ Настройки      ]
```

**Логика:**
- Строка 1: Два самых частых отчёта — текущий и прошлый месяц. Один тап.
- Строка 2: Записи (последние 10) + поиск. Покрывает 90% запросов по транзакциям.
- Строка 3: Баланс (взносы/расчёты) + настройки. Баланс важен для семьи.

**Что убрано из корня:**
- "📁 Конверты" → переехал в Настройки (используется редко)
- "❓ Помощь" → убрана кнопка (пользователь может просто написать "помощь")
- "📅 Эта неделя" → убрана (пользователь может написать "расходы за неделю")
- "📈 Тренды" → убрана кнопка (доступно через свободный текст)
- "🗓 Другой период" → убрана кнопка (пользователь вводит "отчёт за февраль")

**Подменю "⚙️ Настройки":**

```
[ 🌍 Язык   ]  [ 📁 Конверт ]
[ ↩️ Отмена ]  [ 🔧 Админ ▸  ]  ← только для admin
```

**Подменю "🔧 Админ" (только admin):**

```
[ ⚙️ Конфиг    ]  [ 👥 Юзеры   ]
[ 🧠 База знаний ]  [ 🔄 Обновить ]
```

Итого: максимум 2 тапа до любой функции вместо 3-4.

### Навигация:

Каждое подменю имеет "← Назад" кнопку. Корневой уровень — без "Назад".

## DASHBOARD v2

### Проблема текущего Dashboard:
- Формат: свободный текст в ячейках → бот не может парсить
- Нет баланса пользователей в сводке
- Нет структурированных ключей для программного чтения

### Новая структура — 4 секции:

#### Секция A: SNAPSHOT (строки 1-12)
Ключ-значение, бот читает по ключу.

| Ключ | Значение | Описание |
|------|----------|----------|
| month | 2026-04 | Текущий месяц |
| budget | 5000.00 | Бюджет (cap) |
| spent | 2341.50 | Расходы |
| remaining | 2658.50 | Остаток |
| pct_used | 46.8 | % использовано |
| pace | on_track | on_track / over_pace / under_pace |
| currency | EUR | Базовая валюта |
| days_left | 21 | Дней до конца месяца |
| daily_avg | 78.05 | Средний расход/день |
| daily_budget | 126.60 | Допустимый расход/день |
| updated_at | 2026-04-09 14:30 UTC | Последнее обновление |

**Формат:** Столбец A = ключ, столбец B = значение. Программно читается как dict.

#### Секция B: USER_BALANCE (строки 14-20)
Таблица с заголовками.

| User | Contributed | Share | Balance | Status |
|------|------------|-------|---------|--------|
| Mikhail | 3500.00 | 1170.75 | +2329.25 | surplus |
| Maryna | 0.00 | 1170.75 | -1170.75 | deficit |

**Бот читает:** Balance для каждого юзера → отвечает на "сколько я должна?"

#### Секция C: CATEGORIES (строки 22-35)
Топ категорий текущего месяца.

| Category | Amount | Pct | Count |
|----------|--------|-----|-------|
| Food | 890.50 | 38.0 | 23 |
| Housing | 650.00 | 27.8 | 3 |
| Transport | 340.00 | 14.5 | 8 |
| ... | ... | ... | ... |

**Бот читает:** для быстрых ответов типа "на что больше всего трачу?"

#### Секция D: HISTORY (строки 37+)
Месяц-за-месяцем: spent, budget, balance per user. Для трендов.

| Month | Spent | Budget | Pct | Mikhail_Balance | Maryna_Balance |
|-------|-------|--------|-----|-----------------|----------------|
| 2026-01 | 4200 | 5000 | 84% | +1200 | -1200 |
| 2026-02 | 3800 | 5000 | 76% | +950 | -950 |
| 2026-03 | 4600 | 5000 | 92% | +180 | -180 |

### Как бот использует Dashboard:

Новый tool `get_dashboard` (или обновлённый `get_budget_status`) читает секцию A как словарь → мгновенный ответ без пересчёта всех транзакций. Секция B → баланс. Секция C → категории. Секция D → тренды.

### Как человек использует Dashboard:

Открывает вкладку Dashboard в Google Sheets → видит:
- Сводку бюджета с цветовой индикацией (conditional formatting)
- Кто сколько должен
- Разбивку по категориям
- Историю по месяцам
