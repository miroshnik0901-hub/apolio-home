## Bugs Found

### BUG-1: Undo (Скасувати) — no visible feedback [MEDIUM]
**Steps:** Menu → Система → Скасувати
**Expected:** Confirmation message ("Останній запис скасовано" or "Немає що скасувати")
**Actual:** Button click produces no response at all. No new message, no edit, no error.
**Impact:** User has no idea if undo worked or not. Tested after saving "кава 3.50" — transaction still in DB after clicking Скасувати multiple times.
**Verdict:** Either the handler is broken or the callback_data doesn't match the dispatch.

---

## UI/UX Issues

### UX-1: Mixed languages in admin messages [LOW]
When UI language is set to Ukrainian (🇺🇦), several admin messages display in Russian:
- Config view: "Открыть в Google Sheets", "Настройки конверта" → should be "Відкрити в Google Sheets", "Налаштування конверта"
- Knowledge base: "База знаний", "Данных пока нет" → should be "База знань", "Даних поки немає"
- Dashboard refresh: "Дашборд обновлён" → should be "Дашборд оновлено"
**Root cause:** These strings are likely hardcoded in Russian and not routed through `i18n.ts()`.

### UX-2: Оновити меню — no confirmation [LOW]
**Current:** Button executes silently, redirects to Settings submenu.
**Suggested:** Show brief confirmation like "✅ Меню оновлено" before redirect.
